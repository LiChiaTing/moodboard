Warm Studio

The shared visual language for our product — warm, friendly, and confident. Use these tokens, type styles, and components so every screen feels like the same product. This is a living reference; if a screen breaks a rule, that's a conversation, not a quiet exception.

1

Personality

Five principles guide every decision. When in doubt, choose the option that feels more warm and human, not more technical.

🌿  Warm, not white  — cream canvas, never clinical pure-white pages.
💪  Bold but friendly  — heavy headlines softened by rounded shapes.
🎨  Playful color, calm base  — neutral base, color in small joyful doses.
🫧  Soft everywhere  — rounded corners and pills, no hard edges.
🎯  One clear action  — a single obvious next step per screen.

💡 Tiles drop from 3-up to 2-up on mobile.
  • 📐One columnTwo-column desktop layouts stack vertically — visual on top, controls below.
  • ↔️Tighter paddingScreen edges go from 80–100px down to 20–24px. Cards from 28–36px to 16–20px.
  • 🔡Headlines shrink, body holdsHero 64→38px, but body never drops below 14px. Keep the 900 weight.
  • 👆44px touch targetsEvery tappable thing gets a ≥44×44px hit area, with ≥8px between them.
  • ⬇️Sticky bottom CTAThe primary action becomes a full-width sticky bar — always in thumb's reach.
  • ✋No hover relianceHover is decorative on touch. Add an :active press (scale 0.97) for feedback.

Breakpoints

Mobile≤ 600pxSingle column, full-bleed sections.
Tablet601–1024pxOne or two columns depending on content.

↔️ This very page is responsive — drag your browser narrow (or open it on your phone) to watch everything restack and rescale.

How to use these tokens: the full written spec lives in DESIGN_SYSTEM.md in this repo. Copy the :root token block into your global CSS and always reference var(--token) — never hard-code a hex value.

Questions or a screen that doesn't fit the rules? Bring it up — we update the system, we don't quietly break it.
