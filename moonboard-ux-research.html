Moodboard — UX Research

Cascadia AI Hackathon 2026 · 3 User Personas + Journey Map · Dianne Ting, sole designer
Core product capabilities
① Read the space
AI analyses the user's uploaded room photo — detecting existing colours, furniture, lighting, and layout.
② Understand style intent
Visual style tiles + budget input let the AI infer what the user actually wants — without asking them to describe it.
③ Web search for products
AI searches the web in real time for products that match the detected style, palette, and budget — not a fixed database.
④ Recommend a shopping list
Curated product cards with prices, purchase links, and a "why this fits" reason for each item. Running budget tracker included.
★ ⑤ Composite into room photo
AI places the recommended products into the user's actual uploaded photo — so they can see how the room would look before buying anything.
⑥ Export: photo + shopping list
Final output: the synthesised room photo + full shopping list as a downloadable PDF or shareable link. The plan they can return to.
End-to-end flow:   Upload photo  →  Set budget  →  Tap style tiles  →  AI reads space + searches web  →  See products in your actual room (composited photo)  →  Save photo + shopping list

Core design principles
1 — Show, don't ask
Users can't articulate what they want. Every input must be visual and reactive — never an open text prompt. Budget is set by a slider, not a form field.
2 — Cohesion is the promise
The core pain is mismatch. Every product card should reinforce "these go together" — not just individual picks. Cohesion score % for Alex; starter-kit framing for Jordan.
3 — Save for later is the CTA
None of these users buy immediately on first session. The moodboard PDF + email/share option is the real conversion moment. "Buy now" links are secondary, inside the card.
