# Decision log

Key design and project decisions, with the reasoning behind them. This is also the
raw material for case studies — capture the *why*, not just the *what*.

## Format

For each entry: what was decided, why, and any alternatives considered.

---

## 2026-06-02 — Forked the team project into a personal "Moodboard" repo

**Decision:** Fork the team hackathon repo into a personal repository named
`moodboard`, deploy the frontend, and link it from the portfolio site.

**Why:** To showcase the frontend/design work as a portfolio piece and to have a
space to experiment with the backend (image-generation models, product sourcing)
without affecting the team's work.

**Notes:** `main` holds the polished standalone demo; the team's backend-wired
version is preserved on the `backend-integration` branch for later experimentation.
